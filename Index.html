Semantic HTML5 Structure

July 29, 2025

Why Use Semantic Tags?

Semantic tags provide meaning to your content, helping browsers, screen readers, and search engines understand the layout and purpose of each section.

Key HTML5 Elements

  • <header>: Contains introductory content or navigation links.
  • <nav>: Defines a set of navigation links.
  • <main>: Represents the dominant content of the page.
  • <article>: Encapsulates self-contained, reusable content.
  • <section>: Groups related content.
  • <aside>: Contains content indirectly related to the main content.
  • <footer>: Defines the footer for a document or section.

Accessibility Best Practices

Making Web Content Inclusive

Web accessibility ensures that all users, regardless of ability, can use your website effectively. Proper HTML structure helps screen readers parse content accurately.

Tips for Accessible HTML

  1. Use semantic elements for structure.
  2. Provide descriptive alt text for images.
  3. Ensure content is keyboard navigable.
  4. Use landmarks like role="navigation" or aria-label to assist screen readers.
  5. Maintain proper heading levels (e.g., h1 → h2 → h3).

Basic SEO Principles

Why SEO Matters

SEO (Search Engine Optimization) helps your site appear higher in search engine results, bringing more visibility to your content.

Beginner SEO Techniques

  • Use unique, descriptive title tags and meta descriptions.
  • Structure content with semantic HTML and clear headings.
  • Include keywords naturally in content and headings.
  • Use alt text for images to help with image search indexing.
